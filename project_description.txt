oT Sensor Monitoring & Motor Control System - Project Documentation

1. Project Overview

Project Title

Multi-Node IoT Sensor Monitoring and Motor Control System



Objective

Design and develop a mobile application that monitors data from five different sensors (PH, Electrical Conductivity, MQ2 Gas Sensor, DHT11, and Water Temperature Sensor) and controls a motor based on defined threshold conditions. The system will be scalable to support multiple sensor nodes in a master-slave architecture.



Use Case

This system can be applied in smart agriculture, aquarium monitoring, greenhouse automation, or water quality management where real-time sensor monitoring and automated control is required.

2. System Architecture

High-Level Architecture





[ESP32 Node 1 - Master] ──┐

[ESP32 Node 2 - Slave]  ──┼──> [Firebase Realtime DB] <──> [React Native App]

[ESP32 Node 3 - Slave]  ──┘

Components

Hardware Layer: ESP32 microcontrollers with sensors and relay modules

Backend Layer: Firebase Realtime Database

Application Layer: React Native mobile application

Communication Protocol: WiFi + MQTT/HTTP

3. Hardware Specification

Microcontroller

Device: ESP32 Development Board

Features: Built-in WiFi, Bluetooth, multiple ADC pins, GPIO pins

Power: 5V USB or external power supply

Sensors Used

Physical Sensors (Actual Hardware)

DHT11 Temperature & Humidity SensorMeasures: Air temperature (0-50°C) and humidity (20-90% RH)

Interface: Digital (One-Wire protocol)

Pin: GPIO 4

MQ2 Gas SensorMeasures: LPG, Propane, Methane, Smoke (200-10000 ppm)

Interface: Analog output

Pin: GPIO 32

Simulated Sensors (Software-Based)

PH SensorSimulated Range: 6.0 - 8.0 pH

Generated with realistic noise patterns

Electrical Conductivity (EC) SensorSimulated Range: 200 - 800 μS/cm

Time-based variation for realism

Water Temperature SensorDerived from DHT11 air temperature + offset

Range: 15 - 40°C

Actuator

5V Relay Module: Controls AC/DC motor (or LED for demonstration)

Pin: GPIO 23

Switching Capacity: 10A at 250VAC

Hardware Connections





ESP32 GPIO Pinout:

├── GPIO 4  → DHT11 Data Pin

├── GPIO 32 → MQ2 Analog Output

├── GPIO 23 → Relay Module (Motor Control)

├── 3.3V    → Sensor VCC

└── GND     → Common Ground

Configurable limits for each sensor

Control Logic



IF (ph < phMin OR ph > phMax) OR

   (ec > ecMax) OR

   (gas > gasMax) OR

   (temperature > tempMax)

THEN

   Motor = ON

ELSE

   Motor = OFF

7. ESP32 Firmware Logic

Main Functions

Setup Phase

Initialize WiFi connection

Connect to Firebase

Initialize all sensors

Set relay pin as OUTPUT

Loop Phase (Runs Continuously)

Read DHT11 sensor (temperature, humidity)

Read MQ2 gas sensor (analog value)

Generate simulated PH value

Generate simulated EC value

Calculate water temperature from air temp

Push all data to Firebase under node path

Check Firebase for motor control commands

Apply threshold logic if in AUTO mode

Control relay based on decision

Wait 2-5 seconds, repeat

Code Flow





Start → WiFi Connect → Firebase Auth → Read Sensors 

→ Upload Data → Check Commands → Control Motor → Delay → Loop

9. Communication Flow

ESP32 → Firebase → App

ESP32 reads sensor data

Formats as JSON object

Sends to Firebase path: /nodes/{nodeId}/sensors

Firebase triggers real-time listener in app

App UI updates automatically

App → Firebase → ESP32

User presses motor control button

App writes to Firebase: /nodes/{nodeId}/motor/status

ESP32 listens to this path

ESP32 receives command

ESP32 controls relay module

Motor turns ON/OFF
